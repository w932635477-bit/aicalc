# Contributing

Thanks for your interest in aicalc! Here's how to contribute.

## Quick Start

1. Fork the repo
2. Create a feature branch: `git checkout -b feat/my-feature`
3. Make your changes
4. Run `npm run build` to verify no errors
5. Push and open a Pull Request

## Guidelines

- **One PR per feature** — keep changes focused
- **Test your changes** — run `npm run build` at minimum
- **Match existing style** — follow the code conventions you see in the project
- **No secrets** — never commit API keys, tokens, or credentials

## Reporting Issues

Found a bug? Open an issue with:
- What you expected
- What actually happened
- Steps to reproduce
- Browser and OS
